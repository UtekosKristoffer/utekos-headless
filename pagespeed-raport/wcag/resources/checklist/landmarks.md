# Landmarks

This document summarizes landmark-related accessibility guidance and the
associated WCAG requirement where applicable.

| Topic                                 | Technique                                                                                                                                                                                                                                                                                                                                                                                                                               | Testing Methodology                                                                                         | Requirement Level | WCAG Criterion                                                          |
| ------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------- | ----------------- | ----------------------------------------------------------------------- |
| Landmarks to Bypass Blocks of Content | [Bypass Blocks](https://dequeuniversity.com/class/semantic-structure-wcag-2.2/within-pages/): Screen readers allow users to navigate by landmarks, so landmarks are an effective way to bypass blocks of content, as required by WCAG 2.4.1. Other methods may also be used instead of, or in addition to, landmarks, such as skip links, [headings](https://dequeuniversity.com/checklists/web/headings), and expand/collapse regions. | [Testing methodology 2.4.1.a](https://dequeuniversity.com/class/dequeway/interpreting/wcag/a/2.4.1/2.4.1.a) | Required          | [WCAG 2.4.1](https://www.w3.org/WAI/WCAG22/Understanding/bypass-blocks) |
| Landmark Structural Organization      | [Page Layout Groupings](https://dequeuniversity.com/class/semantic-structure-wcag-2.2/landmarks/creating-landmarks#creating-landmarks): Landmarks SHOULD be used to accurately designate pre-defined parts of the layout, such as the header, navigation, main content, and footer.                                                                                                                                                     | N/A                                                                                                         | Best Practice     | N/A                                                                     |
| Landmark Structural Organization      | [Content Within Landmarks](https://dequeuniversity.com/class/semantic-structure-wcag-2.2/landmarks/best-practices#all-content-in-a-landmark): All text SHOULD be contained within a landmark region.                                                                                                                                                                                                                                    | N/A                                                                                                         | Best Practice     | N/A                                                                     |
| Landmark Structural Organization      | [Landmark Names](https://dequeuniversity.com/class/semantic-structure-wcag-2.2/landmarks/best-practices#distinguishable-among-same-type): Multiple instances of the same type of landmark SHOULD be distinguishable by different discernible names using `aria-label` or `aria-labelledby`.                                                                                                                                             | N/A                                                                                                         | Best Practice     | N/A                                                                     |
| Landmark Structural Organization      | [Only One Instance of Some Landmarks](https://dequeuniversity.com/class/semantic-structure-wcag-2.2/landmarks/best-practices#limit-of-type-per-page): A page SHOULD NOT contain more than one instance of each of the following landmarks: `header` or `banner`, `main`, and `footer` or `contentinfo`.                                                                                                                                 | N/A                                                                                                         | Best Practice     | N/A                                                                     |
| Landmark Structural Organization      | [Limit the Number of Landmarks](https://dequeuniversity.com/class/semantic-structure-wcag-2.2/landmarks/best-practices#total-number): The total number of landmarks SHOULD be minimized to the extent appropriate for the content. Having a large number of landmarks defeats the main purpose of landmarks, which is to make it easy to navigate quickly to sections of the layout.                                                    | N/A                                                                                                         | Best Practice     | N/A                                                                     |
| Markup                                | [Markup](https://dequeuniversity.com/class/semantic-structure-wcag-2.2/landmarks/creating-landmarks#html-landmarks-aria-equivalents): Landmarks MAY be designated with either HTML tags or their equivalent ARIA roles, for example `<header>` or `role="banner"`, `<nav>` or `role="navigation"`, `<main>` or `role="main"`, and `<footer>` or `role="contentinfo"`.                                                                   | N/A                                                                                                         | Best Practice     | N/A                                                                     |
